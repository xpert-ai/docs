# Share

## Overview

Share功能允许您将创建的故事Share给团队成员或其他用户，支持多种Share方式和权限控制。

## Share方式

### 公开链接

生成公开链接，任何人都可以通过链接访问故事：
- 适合公开的数据分析报告
- 无需登录即可访问
- 可以设置访问密码

### 私有链接

生成私有链接，只有有权限的用户才能访问：
- 需要登录验证
- 可以设置Access Permission
- 适合内部团队使用

### 嵌入代码

生成嵌入代码，可以将故事嵌入到其他网站或应用中：
- 支持iframe嵌入
- 可以自定义尺寸
- 保持响应式布局

## Share设置

### Access Permission

- **公开**: 所有用户都可以访问
- **私有**: 仅指定用户可以访问
- **密码保护**: 需要输入密码才能访问

### Share范围

- **所有人**: 任何人都可以访问
- **组织内**: 仅组织内用户可以访问
- **指定用户**: 指定特定用户或用户组

### 有效期设置

- **永久有效**: 链接永久有效
- **限时有效**: 设置链接的有效期
- **访问次数限制**: 限制链接的访问次数

## 协作功能

### 评论功能

团队成员可以在故事中添加评论：
- 针对特定图表添加评论
- @提及其他用户
- 回复评论

### 协作编辑

多人协作Edit Story：
- 实时同步编辑
- 查看其他协作者的操作
- Version History记录

### 通知设置

设置Share和协作的通知：
- 新评论通知
- 编辑通知
- 访问通知

## Share统计

### Access Statistics

查看故事的Share和访问数据：
- Shares
- 访问次数
- 访问来源
- 访问时间分布

### User Feedback

- Comments
- Likes
- 收藏数量

## 最佳实践

1. **权限控制**: 根据数据敏感性设置合适的访问权限
2. **链接管理**: 定期检查和管理分享链接
3. **内容更新**: 更新故事后及时通知分享对象
4. **安全考虑**: 避免分享包含敏感信息的故事
