# 管理语义模型

## 概述

管理语义模型功能允许您创建、编辑、删除和管理数据分析平台中的语义模型。语义模型是数据分析的基础，通过合理的管理可以确保数据分析的准确性和效率。

## 创建语义模型

### 基本步骤

1. 进入语义模型管理页面
2. 点击"新建模型"按钮
3. 填写模型基本信息
   - **模型名称**: 使用清晰、描述性的名称
   - **模型描述**: 详细说明模型的用途和范围
   - **数据源连接**: 选择或配置数据源
4. 配置模型结构
   - **定义维度**: 设置分析的维度，如时间、地区、产品等
   - **定义度量**: 设置要分析的数值指标
   - **设置层次结构**: 配置维度内的层级关系
5. 配置模型参数
   - **模型类型**: 选择SQL多维模型或MDX多维模型
   - **准确度设置**: 调整查询转换的准确度
   - **响应速度**: 平衡准确度和响应速度
6. 保存并发布模型

### 模型配置示例

```json
{
  "name": "销售数据分析模型",
  "description": "用于分析销售数据的多维模型",
  "dataSource": "sales_db",
  "dimensions": ["时间", "地区", "产品"],
  "measures": ["销售额", "订单量", "客户数"]
}
```

## 编辑语义模型

### 修改模型配置

1. 在模型列表中找到要编辑的模型
2. 点击"编辑"按钮进入编辑模式
3. 修改模型配置
   - 更新基本信息
   - 添加或删除维度
   - 修改度量定义
   - 调整层次结构
4. 预览更改效果
5. 保存更改

### 版本管理

系统支持模型版本管理：
- 每次保存都会创建新版本
- 可以查看历史版本
- 支持回滚到之前的版本

## 删除语义模型

> **⚠️ 警告**: 删除模型将同时删除所有相关的查询和分析结果，此操作不可恢复，请谨慎操作。

### 删除步骤

1. 选择要删除的模型
2. 点击"删除"按钮
3. 系统会提示删除影响范围
4. 确认删除操作
5. 输入确认信息完成删除

## 模型权限管理

### 访问权限设置

- **公开**: 所有用户都可以访问和使用
- **私有**: 仅创建者可以访问
- **指定用户**: 指定特定用户或用户组可以访问
- **部门权限**: 按部门设置访问权限

### 权限级别

- **只读**: 只能查看模型和查询结果
- **编辑**: 可以修改模型配置
- **管理**: 可以删除模型和管理权限

## 模型导入导出

### 导出模型

可以将模型配置导出为JSON文件，用于备份或迁移。

### 导入模型

从JSON文件导入模型配置，快速创建新模型。

## 最佳实践

1. **命名规范**: 使用清晰、描述性的模型名称，遵循统一的命名规范
2. **文档说明**: 为每个模型添加详细的说明文档，包括用途、数据源、使用场景等
3. **版本管理**: 定期备份和版本管理，重要更改前先备份
4. **权限控制**: 合理设置模型访问权限，确保数据安全
5. **性能优化**: 合理设计维度结构，避免过度复杂的层次结构
6. **定期审查**: 定期审查模型使用情况，清理不再使用的模型
