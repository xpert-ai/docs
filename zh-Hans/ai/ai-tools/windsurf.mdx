配置 Windsurf 的 Cascade AI 助手，帮助你编写与维护文档。本指南展示如何专为 Mintlify 文档流程设置 Windsurf。

## 前置条件

- 已安装 Windsurf 编辑器
- 可访问你的文档仓库

## 工作区规则

创建工作区规则，为 Windsurf 提供你的文档项目上下文与规范。

在项目根目录创建 `.windsurf/rules.md`：

````markdown
# Mintlify 技术写作规范

## 项目背景

- 这是基于 Mintlify 平台的文档项目
- 我们使用带 YAML Frontmatter 的 MDX 文件
- 导航在 `docs.json` 中配置
- 遵循技术写作最佳实践

## 写作标准

- 说明使用第二人称（“你”）
- 使用主动语态与现在时
- 步骤说明以前置条件开头
- 重要步骤包含预期结果
- 使用描述性、关键词丰富的标题
- 句子简洁但信息充分

## 必要页面结构

每个页面都必须以 frontmatter 开头：

```yaml
---
title: "清晰、具体的标题"
description: "用于 SEO 与导航的简洁描述"
---
```

## Mintlify 组件

### 提示框

- `<Note>` 用于补充说明
- `<Warning>` 用于重要注意事项或破坏性变更
- `<Tip>` 用于最佳实践与专家建议
- `<Info>` 用于中性背景信息
- `<Check>` 用于成功确认

### 代码示例

- 适当时提供完整可运行示例
- 多语言示例使用 `<CodeGroup>`
- 所有代码块标注语言标签
- 使用真实数据而非占位符
- API 文档使用 `<RequestExample>` 与 `<ResponseExample>`

### 操作步骤

- 顺序说明使用 `<Steps>` 组件
- 需要时用 `<Check>` 添加验证步骤
- 将复杂流程拆分为更小步骤

### 内容组织

- 平台差异使用 `<Tabs>`
- 渐进披露使用 `<Accordion>`
- 重点内容使用 `<Card>` 与 `<CardGroup>`
- 图片使用 `<Frame>` 并提供清晰的 alt 文本

## API 文档要求

- 使用 `<ParamField>` 记录全部参数
- 使用 `<ResponseField>` 展示响应结构
- 提供成功与错误示例
- 嵌套对象用 `<Expandable>`
- 始终包含鉴权示例

## 质量标准

- 发布前测试所有代码示例
- 站内链接使用相对路径
- 所有图片包含 alt 文本
- 保持标题层级正确（从 h2 开始）
- 检查既有模式保持一致性
````
